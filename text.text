В дополнение к моему основному описанию учитывай, что проект использует следующий стек:

Frontend: React 18.2.0 с TypeScript 5.3.3, Vite 5.0.11, React Router DOM 6.21.3, UI-библиотеки shadcn/ui (Radix UI компоненты), Tailwind CSS 3.4.1 с tailwindcss-animate, редактор контента TipTap 3.10.5 с расширениями, Zustand и TanStack React Query для управления состоянием, GraphQL (graphql-request) как основной API клиент и другие вспомогательные библиотеки.

Backend: KeystoneJS 6 с TypeScript 5 и Node.js 18-22, база данных SQLite (better-sqlite3) с поддержкой MySQL и PostgreSQL через Prisma ORM, реализующая аутентификацию через OAuth2 (Passport.js) и JWT, использует Helmet для безопасности, express-rate-limit для rate limiting, cors для CORS, Redis (ioredis) для кэширования и сессий, Winston для логирования. GraphQL API как основной интерфейс (REST опционально).

Требования к backend:

Для логгирования используй утилиту logger (Winston).

Используй встроенные механизмы KeystoneJS для ролей, прав доступа (RBAC), OAuth2 (через Passport.js) и JWT аутентификацию.

Полная валидация и санитаризация всех входящих данных через Zod (встроен в KeystoneJS) и express-validator, без самодельных костылей.

Отказоустойчивый, с обработкой ошибок, логированием и защитой от XSS, CSRF, SQL/NoSQL-инъекций, brute-force.

Настрой CORS, rate limiting, HTTPS и Content Security Policy для безопасности API.

Предоставь ясные GraphQL API эндпоинты, которые можно безопасно и удобно использовать с фронтендом (React), включая GraphQL queries, mutations и helper-функции. KeystoneJS автоматически генерирует GraphQL схему на основе определённых схем данных.

Используй официальные плагины и middleware, избегай изобретения велосипеда.

Обеспечь документацию к GraphQL API и настройкам безопасности для frontend-разработчиков.

Учитывай разделение frontend (frontend-react/) и backend (backend/keystonejs-backend/) в монолитном репозитории, проксирование запросов через Vite (/api → http://localhost:1337).

Решай проблемы без технических долгов, колхозных решений и костылей.

Перед решением задачи изучи интернет-ресурсы, учтывай стек и очень внимательно прочитай документацию.

Когда создаешь схемы данных в KeystoneJS, сразу расставляй индексы через Prisma и все необходимые оптимизации для бесперебойной, стабильной и быстрой работы в нагруженных условиях.
