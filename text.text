В дополнение к моему основному описанию учитывай, что проект использует следующий стек:

Frontend: React 18.2.0 с TypeScript 5.3.3, Vite 5.0.11, React Router DOM 6.21.3, UI-библиотеки shadcn/ui (Radix UI компоненты), Tailwind CSS 3.4.1 с tailwindcss-animate, редактор контента TipTap 3.10.5 с расширениями, Zustand и TanStack React Query для управления состоянием, Axios как HTTP клиент и другие вспомогательные библиотеки.

Backend: Strapi 5.30.0 с TypeScript 5 и Node.js 18-22, база данных SQLite (better-sqlite3), с поддержкой MySQL и PostgreSQL, реализующая аутентификацию через OAuth2 и JWT, использует Helmet для безопасности, koa-ratelimit для rate limiting, @koa/cors для CORS, Redis (ioredis) для кэширования и сессий.

Требования к backend:

Используй встроенные механизмы Strapi для ролей, прав доступа (RBAC), OAuth2 и JWT аутентификацию.

Полная валидация и санитаризация всех входящих данных, без самодельных костылей.

Отказоустойчивый, с обработкой ошибок, логированием и защитой от XSS, CSRF, SQL/NoSQL-инъекций, brute-force.

Настрой CORS, rate limiting, HTTPS и Content Security Policy для безопасности API.

Предоставь ясные API эндпоинты, соответствующие REST, которые можно безопасно и удобно использовать с фронтендом (React/Next.js), включая API helper-функции.

Используй официальные плагины и middleware, избегай изобретения велосипеда.

Обеспечь документацию к API и настройкам безопасности для frontend-разработчиков.

Учитывай разделение frontend (frontend-react/) и backend (backend/strapi-backend/) в монолитном репозитории, проксирование запросов через Vite (/api → http://localhost:1337).
